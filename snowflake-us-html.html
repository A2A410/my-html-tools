‚ÑπÔ∏è How It Works
The userscript uses WebRTC to create peer-to-peer connections between censored users and the Tor network. When enabled, it connects to Snowflake's broker server (wss://snowflake-broker.torproject.net) and waits for client assignments. When a censored user needs help, the broker pairs you up and you relay their traffic to a Tor bridge.

Frequently Asked Questions

Is this safe?

Yes. You're only relaying encrypted Tor traffic. You cannot see what users are browsing, and their traffic doesn't exit from your IP address.

Does it use a lot of bandwidth?

No. Each connection typically uses very little bandwidth. Snowflake is designed for minimal resource usage.

Can I leave it running all the time?

Yes! The more volunteers available, the more reliable Snowflake becomes for censored users.

What's the difference from the extension?

Functionally the same. The extension can run in background; the userscript requires a browser tab open.

Where can I see detailed stats?

Click the minimize button on the floating panel to collapse it. Stats are always visible when expanded.

Does it work on mobile?

Only if your mobile browser supports userscripts (e.g., Firefox Mobile with Tampermonkey).
